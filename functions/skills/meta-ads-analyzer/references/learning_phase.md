# Learning Phase

## Overview

The learning phase is the period when Meta's delivery system is gathering data to optimize ad delivery. During this phase, performance is less stable and costs may be higher.

## Entry Conditions

An ad set enters the learning phase when:
- A new ad set is created
- A significant edit is made to an existing ad set

## Significant Edits That Reset Learning

The following changes reset the learning phase:

- **Targeting changes** — Any modification to audience targeting
- **Creative changes** — New ad creative, significant text/image changes
- **Optimization event changes** — Changing what you optimize for
- **Bid strategy changes** — Switching bid strategies
- **Budget changes >20%** — Large budget increases or decreases
- **Adding a new ad** to the ad set
- **Pausing for 7+ days** and restarting

## Exiting the Learning Phase

- Requires approximately **50 optimization events** within a 7-day window
- Once exited, delivery stabilizes and performance becomes more predictable
- The system continues to learn and adapt, but with less volatility

## Learning Limited

An ad set enters "Learning Limited" when:
- It's unlikely to reach 50 optimization events in 7 days
- Common causes: small audience, low budget, too many ad sets splitting budget, low conversion rate

## Best Practices During Learning Phase

1. **Avoid making significant edits** — Let the system gather data
2. **Don't judge performance** — Costs during learning are not representative
3. **Ensure adequate budget** — Budget should support ~50 conversions/week per ad set
4. **Consolidate ad sets** — Fewer, broader ad sets exit learning faster
5. **Use broader optimization events** — If conversions are rare, optimize for a higher-funnel event
